STM32 Betriebsystem

Projektbeschreibung

In diesem Projekt habe ich eine betriebsystemähnliche Software für einen STM32-Mikrocontroller programmiert. Sie kann grundlegende Aufgaben wie GPIO-Steuerung, Interrupt-Verwaltung und Timernutzung bewältigen. Die Implementierung dieser Features erfolgte über Low-Level Zugriffe auf Register, wobei einzelne Bits gesetzt/gelöscht werden mussten. Das Betriebsystem kommt komplett ohne externe Libraries oder Frameworks aus. Das Ziel war es den Ablauf von Programmen auf Mikrocontrollern zu verstehen.

Tools

Hardware: STM32L152RE
Software: C, STM32CubeIDE (nur als Editor/Compiler verwendet)

Herausforderungen

Die größte Herausforderung war das Programmieren ohne HAL oder andere Frameworks. Das erforderte ein genaues Verständnis der Registerstrukturen. Insbesondere das korrekte Setup der ARM Komponente wie NVIC für die Handhabung von Interrupts und Timer waren komplex, aber extrem Lehrreich. So konnte ich lernen wie Mikrocontroller wirklich funktionieren, wenn keine abstrahierenden Libraries verwendet werden.
